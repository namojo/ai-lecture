# 에러 처리와 품질 보증 패턴

## 학습 목표

- AI 에이전트의 에러를 감지하고 복구하는 패턴을 익힌다
- 하네스 파이프라인의 품질 게이트를 설계할 수 있다
- 프로덕션 수준의 안정적인 자동화를 구현할 수 있다

## AI 에이전트의 일반적 실패 유형

| 유형 | 예시 | 대처 |
|------|------|------|
| 구문 에러 | 잘못된 TypeScript 코드 생성 | 빌드 검증 후 재시도 |
| 논리 에러 | 요구사항과 다른 구현 | 리뷰어 에이전트 검증 |
| 누락 | 일부 기능 미구현 | 체크리스트 기반 검증 |
| 할루시네이션 | 존재하지 않는 API 사용 | 빌드 + 타입 체크 |
| 무한 루프 | 같은 수정 반복 | 최대 시도 횟수 제한 |

> [!WARNING] AI는 실패할 수 있다
> AI 에이전트를 "완벽한 개발자"로 가정하면 안 됩니다. 반드시 **검증 단계**를 포함하고, 실패 시 복구 전략을 마련하세요.

## 패턴 1: 빌드 검증 게이트

모든 코드 생성 후 반드시 빌드를 확인합니다:

```markdown
## 오케스트레이터 워크플로우
1. 에이전트에게 코드 생성 요청
2. `npm run build` 실행
3. 빌드 실패 시:
   - 에러 메시지를 에이전트에게 전달
   - "이 빌드 에러를 수정해줘" 요청
   - 최대 3회 재시도
4. 3회 실패 시 중단 + 사람에게 알림
```

```typescript
async function buildGate(maxRetries = 3): Promise<boolean> {
  for (let i = 0; i < maxRetries; i++) {
    const result = await bash("npm run build 2>&1");
    if (result.exitCode === 0) return true;
    await agent("fixer").run(`빌드 에러 수정: ${result.stderr}`);
  }
  return false; // 사람 개입 필요
}
```

## 패턴 2: 체크리스트 기반 리뷰

리뷰어 에이전트에게 구체적 체크리스트를 제공합니다:

```markdown
# reviewer 에이전트 프롬프트

## 검증 항목
- [ ] TypeScript strict 에러 0건
- [ ] 외부 네트워크 요청 0건
- [ ] 모든 컴포넌트에 "use client" 또는 서버 컴포넌트 명시
- [ ] 한국어 텍스트 일관성 (용어집 준수)
- [ ] 접근성: alt 속성, aria-label 포함
```

> [!TIP] 구체적 체크리스트
> "코드를 리뷰해줘"보다 "이 10개 항목을 확인해줘"가 훨씬 정확한 리뷰 결과를 만듭니다.

## 패턴 3: 폴백 전략

```typescript
// 1차: 전문 에이전트 시도
let result = await agent("specialist").run(task);

if (!result.success) {
  // 2차: 범용 에이전트로 폴백
  result = await agent("generalist").run(task);
}

if (!result.success) {
  // 3차: 사람에게 에스컬레이션
  await notify("수동 처리 필요: " + task.description);
}
```

## 패턴 4: 점진적 검증

한번에 모든 것을 만들지 않고, 단계별로 검증합니다:

```text
Step 1: 타입 정의 → 타입 체크 통과?
Step 2: 핵심 로직 → 단위 테스트 통과?
Step 3: UI 컴포넌트 → 빌드 성공?
Step 4: 통합 → 전체 테스트 통과?
```

> [!INFO] 실패 비용 최소화
> 초기 단계에서 실패를 감지할수록 수정 비용이 적습니다. 타입 에러를 Step 1에서 잡는 것이 Step 4에서 잡는 것보다 훨씬 효율적입니다.

## 품질 메트릭

자동화 파이프라인의 품질을 측정하는 기준:

| 메트릭 | 목표 | 측정 방법 |
|--------|------|----------|
| 빌드 성공률 | > 90% | 1차 빌드 통과 비율 |
| 리뷰 통과률 | > 70% | 수정 없이 통과 비율 |
| 평균 재시도 | < 1.5회 | 평균 수정 사이클 수 |
| 에스컬레이션 | < 5% | 사람 개입 비율 |

## 요약

- AI 에이전트는 실패할 수 있으므로 검증 게이트 필수
- 빌드 검증 + 체크리스트 리뷰 + 폴백 전략 조합
- 점진적 검증으로 실패 비용 최소화
- 최대 재시도 횟수를 제한하여 무한 루프 방지
