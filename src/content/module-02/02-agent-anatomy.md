# 에이전트 정의 파일 해부

## 학습 목표

이 챕터를 완료하면 다음을 이해할 수 있습니다:

- `.claude/agents/*.md` 파일의 구조와 각 필드의 역할
- 에이전트 프론트매터(frontmatter)의 주요 설정 항목
- 실무에서 사용할 수 있는 에이전트 정의 작성법

## 에이전트 파일의 위치와 구조

Claude Code에서 에이전트는 프로젝트 루트의 `.claude/agents/` 디렉토리에 마크다운 파일로 정의합니다. 파일명이 곧 에이전트의 식별자가 됩니다.

```
프로젝트/
├── .claude/
│   ├── agents/
│   │   ├── code-reviewer.md      # 코드 리뷰 에이전트
│   │   ├── test-generator.md     # 테스트 생성 에이전트
│   │   └── doc-writer.md         # 문서 작성 에이전트
│   └── settings.json
└── src/
```

## 프론트매터 필드 해설

에이전트 파일은 YAML 프론트매터와 마크다운 본문으로 구성됩니다.

```yaml
---
name: "코드 리뷰어"
description: "PR의 코드 품질, 보안, 성능을 분석하는 에이전트"
model: "claude-sonnet-4-20250514"
tools:
  - Read
  - Grep
  - Glob
  - Bash
disallowedTools:
  - Write
  - Edit
---
```

| 필드 | 필수 | 설명 | 예시 |
|------|------|------|------|
| `name` | O | 에이전트의 표시 이름 | `"코드 리뷰어"` |
| `description` | O | 역할과 목적 요약 | `"PR 코드 품질 분석"` |
| `model` | X | 사용할 LLM 모델 | `"claude-sonnet-4-20250514"` |
| `tools` | X | 허용할 도구 목록 | `[Read, Grep, Bash]` |
| `disallowedTools` | X | 명시적으로 차단할 도구 | `[Write, Edit]` |

> [!TIP] 도구 제한의 중요성
> `disallowedTools`를 활용하면 에이전트가 **읽기 전용**으로만 동작하게 만들 수 있습니다. 리뷰 에이전트가 실수로 코드를 수정하는 것을 원천 차단할 수 있어 안전합니다.

## 마크다운 본문: 시스템 프롬프트

프론트매터 아래의 마크다운 본문이 에이전트의 **시스템 프롬프트**가 됩니다. 역할, 행동 규칙, 출력 형식을 구체적으로 지시합니다.

```markdown
---
name: "코드 리뷰어"
description: "PR의 코드 품질, 보안, 성능을 분석하는 에이전트"
tools: [Read, Grep, Glob]
---

당신은 시니어 백엔드 개발자로서 코드 리뷰를 수행합니다.

## 리뷰 기준
1. **보안**: SQL 인젝션, XSS, 하드코딩된 시크릿 검사
2. **성능**: N+1 쿼리, 불필요한 루프, 메모리 누수 패턴
3. **가독성**: 네이밍 컨벤션, 함수 길이, 주석 적절성

## 출력 형식
- 심각도: critical / warning / info
- 파일명과 라인 번호를 포함할 것
- 수정 제안 코드를 함께 제시할 것
```

> [!WARNING] 프롬프트 길이 주의
> 시스템 프롬프트가 너무 길면 컨텍스트 윈도우를 소모합니다. 핵심 규칙 5-10개 이내로 압축하고, 상세 참조 자료는 스킬의 `references`로 분리하세요.

## 실전 예시: 로그 분석 에이전트

```markdown
---
name: "로그 분석기"
description: "서버 로그를 분석하여 이상 패턴을 탐지하는 에이전트"
tools: [Read, Grep, Glob, Bash]
disallowedTools: [Write, Edit]
---

당신은 삼성 운영 시스템의 로그 분석 전문가입니다.

## 분석 절차
1. 지정된 로그 디렉토리에서 최근 파일을 탐색
2. ERROR/WARN 레벨 로그의 빈도와 패턴을 분석
3. 시간대별 에러 스파이크를 탐지
4. 반복되는 에러 메시지를 그룹핑

## 출력: 상위 5개 에러 패턴 + 권장 조치 사항
```

> [!INFO] 에이전트 vs CLAUDE.md
> `CLAUDE.md`는 프로젝트 전체에 적용되는 전역 규칙이고, 에이전트 파일은 특정 역할에 특화된 규칙입니다. `CLAUDE.md`가 기본으로 적용되고, 에이전트 파일이 추가로 오버라이드합니다.

## 요약

- 에이전트는 `.claude/agents/*.md`에 마크다운 파일로 정의
- 프론트매터에 이름, 설명, 모델, 도구 허용/차단을 설정
- 마크다운 본문이 시스템 프롬프트로 동작하며 역할과 규칙을 기술
- `disallowedTools`로 읽기 전용 에이전트를 만들어 안전성 확보
- 시스템 프롬프트는 간결하게, 상세 자료는 스킬로 분리
