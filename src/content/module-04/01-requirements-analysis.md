# 요구사항 분석

## 학습 목표

- 로그 분석기의 기능적/비기능적 요구사항을 정의할 수 있다
- AI 도구를 활용한 요구사항 분석 프로세스를 이해한다
- 삼성 운영 환경에서의 로그 분석 시나리오를 파악한다

## 왜 로그 분석기인가?

삼성중공업과 삼성SDI의 운영 환경에서는 수십 대의 서버가 초당 수천 줄의 로그를 생성합니다. 장애 발생 시 엔지니어가 수작업으로 로그를 분석하는 것은 시간이 오래 걸리고 패턴을 놓칠 위험이 있습니다.

> [!TIP] PoC의 가치
> 이 PoC는 "AI가 로그를 읽어준다"가 아니라, **하네스를 사용해 분석 도구를 자동으로 만드는 과정** 자체를 보여줍니다. 도구 자체보다 도구를 만드는 방법이 핵심입니다.

## 기능적 요구사항

| ID | 요구사항 | 우선순위 |
|----|---------|---------|
| FR-01 | 3종 로그 포맷 자동 인식 (syslog, app, access) | 필수 |
| FR-02 | 로그 레벨별 분류 (INFO, WARN, ERROR, FATAL) | 필수 |
| FR-03 | 에러 급증(spike) 탐지 | 필수 |
| FR-04 | 반복 에러 패턴 감지 | 필수 |
| FR-05 | 로그 공백(서비스 중단 의심) 탐지 | 선택 |
| FR-06 | 시각화 차트 (시간대별 에러 빈도) | 필수 |
| FR-07 | 파일 업로드 지원 | 선택 |

## 비기능적 요구사항

```text
성능: 1만 줄 로그 분석 2초 이내
보안: 모든 처리는 브라우저 내 (서버 전송 없음)
호환: 폐쇄망 환경에서 완전 동작
UI: 직관적인 3-탭 구조 (요약/타임라인/원본)
```

## 로그 포맷 분석

분석기가 지원해야 하는 3가지 로그 포맷:

### Syslog 포맷

```text
Mar 15 08:01:08 web-server-01 app-service[3072]: ERROR Database connection timeout
```

### 애플리케이션 로그 포맷

```text
2025-03-15 08:07:00.100 [ERROR] [CacheManager] Redis connection lost: ECONNREFUSED
```

### Access Log 포맷

```text
192.168.1.104 - - [15/Mar/2025:08:02:15 +0900] "POST /api/auth/login HTTP/1.1" 401 89
```

> [!INFO] 정규식 기반 파싱
> 각 포맷은 정규식으로 파싱됩니다. AI가 정규식을 자동 생성하지만, 엣지 케이스 처리를 위해 반드시 개발자가 검증해야 합니다.

## 이상 탐지 알고리즘 개요

1. **에러 급증(Spike)**: 분 단위 에러 수가 평균 + 2σ 초과 시 탐지
2. **반복 패턴**: 동일 에러 메시지가 3회 이상 발생 시 클러스터링
3. **시간 공백**: 2분 이상 로그가 없으면 서비스 중단 의심
