# 대시보드 설계

## 학습 목표

- 운영 대시보드의 핵심 구성 요소를 정의할 수 있다
- 실시간 데이터 시각화의 설계 원칙을 이해한다
- 메트릭 카드, 시계열 차트, 알림 패널의 역할을 파악한다

## 운영 대시보드란?

운영 대시보드는 서버 인프라의 실시간 상태를 한눈에 보여주는 도구입니다. 삼성중공업과 삼성SDI의 운영팀은 이를 통해 장애를 조기에 감지하고 대응합니다.

## 핵심 구성 요소

| 영역 | 목적 | 업데이트 주기 |
|------|------|-------------|
| 메트릭 카드 (4개) | CPU/Memory/Disk/Network 현황 | 1초 |
| 시계열 차트 (4개) | 5분 롤링 윈도우 추이 | 1초 |
| 알림 패널 | 임계값 초과 알림 | 이벤트 발생 시 |
| 서비스 상태 | 6개 서비스 헬스 체크 | 5초 |

> [!TIP] 정보 계층
> 대시보드는 **글랜스(glance) → 스캔(scan) → 상세(detail)** 순으로 정보를 제공해야 합니다. 메트릭 카드로 1초 만에 현황 파악, 차트로 추이 확인, 알림으로 상세 내용 확인.

## 임계값 설정

```typescript
const THRESHOLDS = {
  cpu:     { warning: 60, critical: 80 },
  memory:  { warning: 70, critical: 85 },
  disk:    { warning: 80, critical: 90 },
  network: { warning: 60, critical: 80 },
};
```

## 색상 코딩 원칙

- **초록 (#22C55E)**: 정상 범위
- **노랑 (#F59E0B)**: 경고 (warning 임계값 이상)
- **빨강 (#EF4444)**: 위험 (critical 임계값 이상)

메트릭 값에 따라 카드와 차트 색상이 자동으로 변합니다.

> [!WARNING] 색상만 의존하지 말 것
> 색맹 사용자를 위해 숫자 값과 상태 텍스트를 반드시 함께 표시합니다. 색상은 보조적 정보일 뿐입니다.

## 시뮬레이션 모드

실제 서버가 없는 교육 환경이므로, 결정론적 시뮬레이터로 현실적인 메트릭 패턴을 생성합니다:

- **CPU**: 기본 30%, 주기적 배치 스파이크 (70-95%)
- **Memory**: 점진적 증가 (메모리 릭 시뮬레이션), 주기적 GC
- **Disk**: 느린 증가, 로그 로테이션 시 감소
- **Network**: 랜덤 버스트, 트래픽 피크 패턴

> [!INFO] 시드 기반 RNG
> 같은 시드값을 사용하면 매번 동일한 패턴이 생성됩니다. 이는 데모의 재현성(reproducibility)을 보장하며, 교육 시 일관된 경험을 제공합니다.

## 요약

- 4개 메트릭 카드 + 4개 시계열 차트 + 알림 + 서비스 상태
- 임계값 기반 3단계 색상 코딩 (초록/노랑/빨강)
- 시드 기반 결정론적 시뮬레이터로 현실적 패턴 재현
