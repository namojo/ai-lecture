# Claude Code vs Gemini CLI 비교

## 학습 목표

- 두 AI 개발 도구의 아키텍처 차이를 이해한다
- 각 도구의 최적 사용 시나리오를 판단할 수 있다
- 삼성 실무 환경에서의 도구 선택 기준을 수립할 수 있다

## 아키텍처 비교

| 항목 | Claude Code | Gemini CLI |
|------|-----------|------------|
| 개발사 | Anthropic | Google |
| 실행 환경 | 터미널 (프로젝트 내) | 터미널 (독립) |
| 파일 접근 | 자동 (프로젝트 전체) | 수동 (@파일 참조) |
| 코드 실행 | bash 명령 직접 실행 | 불가 |
| 파일 수정 | 직접 수정/생성 | 코드 출력 (수동 적용) |
| 하네스 | 에이전트 + 스킬 시스템 | 제한적 |

## 강점 비교

### Claude Code의 강점

```bash
# 프로젝트 전체 컨텍스트를 이해
claude "이 프로젝트에서 사용하지 않는 import를 모두 정리해줘"
# → 모든 파일을 스캔하고 직접 수정
```

- **프로젝트 컨텍스트**: 전체 파일 구조를 자동으로 파악
- **직접 실행**: 파일 수정, 테스트 실행, 빌드 확인까지 자동
- **하네스 시스템**: 에이전트/스킬로 복잡한 자동화 가능
- **연속 작업**: 여러 단계를 순차적으로 자동 처리

### Gemini CLI의 강점

```bash
# 빠른 코드 생성과 설명
gemini "React에서 useReducer vs useState 언제 써야 하는지 비교해줘"
# → 상세한 설명과 예시 코드
```

- **빠른 응답**: 짧은 질문에 빠른 답변
- **풍부한 설명**: 코드와 함께 상세한 배경 지식 제공
- **독립적 사용**: 프로젝트와 무관한 일반 질문에 적합
- **Google 생태계**: Google Cloud 관련 작업에 강점

## 시나리오별 도구 선택

| 시나리오 | 추천 도구 | 이유 |
|---------|----------|------|
| 새 기능 구현 | Claude Code | 기존 코드 패턴에 맞춰 구현 |
| 버그 디버깅 | Claude Code | 에러 로그 분석 + 파일 수정 |
| 개념 학습 | Gemini CLI | 상세한 설명과 예시 |
| 리팩토링 | Claude Code | 전체 파일 영향도 파악 |
| 알고리즘 설계 | 둘 다 | 여러 접근법 비교 |
| 코드 리뷰 | Claude Code | 프로젝트 맥락 이해 |
| 빠른 프로토타입 | Gemini CLI | 독립적인 코드 조각 생성 |

> [!TIP] 하이브리드 전략
> 실무에서는 한 가지 도구만 고집하지 마세요. Gemini CLI로 아이디어를 탐색하고, Claude Code로 프로젝트에 통합하는 2단계 접근이 효율적입니다.

## 삼성 폐쇄망 환경 고려사항

| 항목 | Claude Code | Gemini CLI |
|------|-----------|------------|
| 오프라인 사용 | 불가 (API 필요) | 불가 (API 필요) |
| 온프레미스 배포 | 별도 협의 필요 | Google Cloud 기반 |
| 데이터 보안 | API로 코드 전송 | API로 코드 전송 |

> [!WARNING] 보안 정책 확인
> 두 도구 모두 코드를 외부 API로 전송합니다. 삼성의 소스 코드 보안 정책에 따라 사용 범위가 제한될 수 있으므로, **반드시 보안팀과 협의** 후 사용하세요.

## 요약

- Claude Code: 프로젝트 맥락 이해 + 직접 실행이 핵심 강점
- Gemini CLI: 빠른 응답 + 풍부한 설명이 핵심 강점
- 실무에서는 하이브리드 접근이 가장 효과적
- 폐쇄망 환경에서는 보안 정책 검토가 선행 필수
