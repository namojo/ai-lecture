## Module 1: AI 기반 개발의 새로운 패러다임 (85분)

AI 기반 개발이 무엇인지 이해하고, Claude Code와 Gemini CLI를 직접 체험한다.

| # | 챕터 | 유형 | 시간 |
|---|-------|------|------|
| 1.1 | AI가 바꾸는 개발자의 일상 | 개념 | 15분 |
| 1.2 | Claude Code: 설치부터 첫 대화까지 | 개념+데모 | 20분 |
| 1.3 | Gemini CLI: 설치부터 첫 대화까지 | 개념+데모 | 20분 |
| 1.4 | 실습: 첫 번째 자동화 스크립트 만들기 | 실습 | 30분 |

### 1.1 AI가 바꾸는 개발자의 일상
- AI 기반 개발이란? 코드 생성이 아닌 협업 모델
- 전통적 IDE 워크플로우 vs AI 증강 워크플로우 비교
- 생산성 향상 실제 사례
- 삼성 맥락: 제조 운영, 선단 관리, 품질 관리에서의 적용 가능성
- 보안, 정확성, 사내 정책 관련 우려 해소

### 1.2 Claude Code: 설치부터 첫 대화까지
- 설치: `npm install -g @anthropic-ai/claude-code`
- 첫 대화, 도구 루프(read-edit-run) 이해
- 데모: Python 스크립트 분석 요청, 트랜스크립트 공유
- 핵심 명령어와 워크플로우

### 1.3 Gemini CLI: 설치부터 첫 대화까지
- 설치, 인증 설정
- 첫 대화, Claude Code와의 차이점
- 데모: Express 서버 스캐폴딩
- 두 도구의 접근 방식 비교

### 1.4 실습: 첫 번째 자동화 스크립트 만들기
- 시나리오: 장비 센서 CSV 데이터 정리
- Claude Code로 Python 스크립트 작성 (단계별 따라하기)
- 데이터 유효성 검증, 요약 리포트 생성

## Module 2: 하네스 시스템의 이해 (125분)

에이전트, 스킬, 오케스트레이터의 개념을 이해하고, 기존 하네스 설정을 읽고 해석할 수 있다.

| # | 챕터 | 유형 | 시간 |
|---|-------|------|------|
| 2.1 | 하네스란 무엇인가: 에이전트, 스킬, 오케스트레이터 | 개념 | 20분 |
| 2.2 | 에이전트 정의 파일 해부 | 개념+데모 | 20분 |
| 2.3 | 스킬 파일 해부 | 개념+데모 | 20분 |
| 2.4 | 아키텍처 패턴: 파이프라인부터 계층적 위임까지 | 개념 | 25분 |
| 2.5 | 실습: 나만의 첫 스킬 만들기 | 실습 | 40분 |

### 2.1 하네스란 무엇인가
- 하네스 = AI 팀의 조직도 (비유)
- 에이전트 = 누가 (전문가 페르소나)
- 스킬 = 어떻게 (절차적 워크플로우)
- 오케스트레이터 = 프로젝트 매니저
- 파일 구조: `.claude/agents/`, `.claude/skills/`

### 2.2 에이전트 정의 파일 해부
- 실제 에이전트 파일 워크스루 (worldbuilder 예제)
- 프론트매터, 역할, 원칙, I/O 프로토콜
- 팀 커뮤니케이션 섹션, 에러 핸들링

### 2.3 스킬 파일 해부
- 실제 스킬 파일 워크스루 (module-writer 예제)
- 프론트매터 트리거링, 점진적 공개
- references/ 디렉토리, 번들 스크립트

### 2.4 아키텍처 패턴 6종
각 패턴에 삼성 관련 비유:
1. **파이프라인**: 제조 조립 라인 → 순차 의존성
2. **팬아웃/팬인**: QA 검사 스테이션 → 병렬 독립 작업
3. **전문가 풀**: 헬프데스크 → 상황 기반 라우팅
4. **생산자-리뷰어**: 도면 검토 프로세스 → 생성 후 검증
5. **감독자**: 교대 근무 감독관 → 동적 작업 할당
6. **계층적 위임**: 조직도 → 재귀적 분해

### 2.5 실습: 나만의 첫 스킬 만들기
- 로그 파일 → 요약 리포트 스킬 작성
- skill.md 작성, references/ 디렉토리 생성
- Claude Code로 테스트

## Module 3: 실전 도구 만들기 (115분)

하네스 개념을 실제 도구 빌드에 적용한다. 스펙에서 작동하는 도구까지의 전체 라이프사이클을 경험한다.

| # | 챕터 | 유형 | 시간 |
|---|-------|------|------|
| 3.1 | 프로젝트 스펙 작성법 | 개념+데모 | 25분 |
| 3.2 | 에이전트 팀 구성 전략 | 개념 | 20분 |
| 3.3 | 오케스트레이터 설계와 데이터 흐름 | 개념+데모 | 25분 |
| 3.4 | 실습: 2인 팀 하네스로 코드 리뷰 도구 만들기 | 실습 | 45분 |

### 3.1 프로젝트 스펙 작성법
- project-spec-writer 스킬 케이스 스터디
- XML 스키마 워크스루
- 스펙 → 작동하는 앱 과정
- 데모: "빌드 상태 알림" 도구 미니 스펙 작성

### 3.2 에이전트 팀 구성 전략
- 에이전트 분리 기준: 전문성, 병렬성, 컨텍스트, 재사용성
- 팀 사이즈 가이드라인
- Agent Team vs Sub-agent 모드 결정

### 3.3 오케스트레이터 설계와 데이터 흐름
- 오케스트레이터 템플릿 워크스루
- TeamCreate, TaskCreate, SendMessage 사용법
- 데이터 전달 전략 (메시지, 태스크, 파일)
- 에러 핸들링 패턴

### 3.4 실습: 2인 팀 하네스로 코드 리뷰 도구 만들기
- security-reviewer + style-reviewer 에이전트 구성
- 오케스트레이터 스킬 작성
- 샘플 코드베이스에서 실행

## Module 4: PoC 1 - 서버 로그 분석기 (180분)

Claude Code와 하네스 자동화를 활용하여 완전한 로그 분석 도구를 처음부터 빌드한다.

| # | 챕터 | 유형 | 시간 |
|---|-------|------|------|
| 4.1 | 요구사항 분석: 서버 운영팀이 필요로 하는 것 | 개념 | 15분 |
| 4.2 | 로그 파싱 엔진 설계 | 개념+데모 | 25분 |
| 4.3 | 이상 탐지 알고리즘 구현 | 개념+데모 | 30분 |
| 4.4 | 리포트 생성과 시각화 | 데모 | 25분 |
| 4.5 | 실습: 나만의 로그 분석기 확장하기 | 실습 | 45분 |
| 4.6 | 하네스로 전체 파이프라인 자동화 | 데모+실습 | 40분 |

### 4.1 요구사항 분석
- 시나리오: 10+ 서버, 기가바이트 로그, 수동 모니터링 한계
- 요구사항: 다중 포맷 파싱, 에러 스파이크 감지, 패턴 클러스터링, 일일 리포트

### 4.2 로그 파싱 엔진 설계
- 정규식 기반 로그 파싱
- 타임스탬프 정규화, 심각도 분류
- Claude Code로 파서 인터랙티브 빌드 (라이브 데모)
- 임베디드 Log Analyzer 컴포넌트 시연

### 4.3 이상 탐지 알고리즘 구현
- 통계적 이상 탐지: 롤링 윈도우 평균, 표준 편차 임계값
- 에러율 스파이크 감지
- 문자열 유사도 기반 패턴 클러스터링
- 모든 로직 TypeScript, 브라우저 내 실행

### 4.4 리포트 생성과 시각화
- Recharts: 에러 타임라인, 심각도 분포, Top 에러 패턴
- 구조화된 HTML/마크다운 리포트 생성

### 4.5 실습: 나만의 로그 분석기 확장하기
- 커스텀 로그 포맷 파서 추가
- 새로운 이상 탐지 규칙 추가
- 리포트 템플릿 커스터마이징

### 4.6 하네스로 전체 파이프라인 자동화
- 파이프라인 패턴: log-collector → parser → analyzer → reporter
- 오케스트레이터 조율 시연
- 학생들이 하네스 구성

- "Active Connections" 또는 "Queue Depth" 메트릭 추가

## Module 6: 고급 패턴과 실전 적용 (120분)

고급 하네스 패턴을 학습하고, 실제 삼성 프로젝트에 적용하는 방법을 설계한다.

| # | 챕터 | 유형 | 시간 |
|---|-------|------|------|
| 6.1 | 복합 패턴: 실전에서의 하네스 조합 | 개념 | 20분 |
| 6.2 | 에러 핸들링과 품질 보장 패턴 | 개념+데모 | 25분 |
| 6.3 | 폐쇄망 환경에서의 AI 도구 운용 전략 | 개념 | 20분 |
| 6.4 | 삼성 업무에 적용하기: 시나리오별 설계 | 워크숍 | 40분 |
| 6.5 | 다음 단계: 자체 하네스 생태계 구축 로드맵 | 개념 | 15분 |

### 6.1 복합 패턴
- Pipeline + Fan-out, Fan-out + Producer-Reviewer, Supervisor + Expert Pool
- 실제 사례: 다국어 문서 파이프라인, 풀스택 앱 개발 팀

### 6.2 에러 핸들링과 품질 보장 패턴
- 하네스 레벨 에러 핸들링: 재시도 정책, 부분 실패 전략
- 데이터 충돌 해결
- 품질 게이트: 자동 검사 후 사람 리뷰
- review-check 스킬 패턴

### 6.3 폐쇄망 환경에서의 AI 도구 운용 전략
- 삼성 핵심: 에어갭/폐쇄망 네트워크
- 로컬 모델 옵션, 오프라인 문서 캐싱, 프록시 설정
- 인터넷 없이 동작하는 하네스 설정 준비

### 6.4 삼성 업무에 적용하기: 시나리오별 설계 (워크숍)
학생들이 삼성 특정 시나리오에 대해 하네스 아키텍처를 설계:
- **시나리오 A**: 자동화 빌드/테스트 파이프라인 알림 시스템
- **시나리오 B**: 장비 센서 데이터 이상 탐지
- **시나리오 C**: 코드에서 내부 문서 자동 생성
- 발표 및 피드백

### 6.5 다음 단계: 자체 하네스 생태계 구축 로드맵
- 팀 전체 하네스 라이브러리 구축
- 프로젝트 간 스킬 공유
- 하네스 설정 버전 관리
- 개인 생산성 → 조직 변혁으로의 경로
